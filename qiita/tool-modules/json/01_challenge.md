# json を作った理由。読みたいのは“構造”だった

API のレスポンス、設定ファイル、ログの断片など、JSON を見る場面は多いです。
ただ、実際に知りたいのは文字列そのものではなく、そこにどんな構造が入っているかです。

json ツールは、その“構造を見る”作業を早くするために作りました。
単なる整形ではなく、修復、比較、変換までをひとつの作業台にまとめることを目指しています。

## 何がつらかったか

- ちょっと壊れた JSON を直したい
- 似たレスポンスの差分を見たい
- 一部だけ抜き出して確認したい
- 毎回別のツールを開きたくない

こういう作業は、頻度は高いのに、それぞれが細かく分かれています。
json は、この細切れの作業をひとつの場所に寄せるためのものです。

## このツールの狙い

“JSON を眺める”のではなく、“JSON を扱う”ための道具にすること。
これが json の中心にある考え方です。
