# json の UI 設計。編集・修復・変換を 1 ページにまとめる

json ツールでは、編集・修復・変換を別ページに分けすぎないことを重視しています。
利用者は、壊れた JSON を直したあと、そのまま比較や抜き出しに進むことが多いからです。

## エディタ中心に置く

まずはエディタが主役です。
内容を貼り、すぐ見て、必要なら直す。
この流れが自然に見えるように、画面の中心をエディタに寄せています。

## 機能を分けるが、散らしすぎない

- 修復は修復
- 比較は比較
- 変換は変換

と責務は分けますが、画面としてはひとつの作業台に見えるようにしています。
これにより、操作の文脈を失いにくくなります。

## 作業の途中で迷わない

JSON を扱うときは、何度も同じデータを確認することがあります。
そのため、最初の入力から最終確認までの距離を短くすることが重要です。
json は、その距離をできるだけ短くするための構成でまとめています。
