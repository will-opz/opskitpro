# json の使い方。貼る、直す、比べる、抜き出す

json は、日常の開発でよくある「とりあえず中身を見たい」を素早く終わらせるためのツールです。
大きな設定も、API レスポンスも、まずはここに貼って扱います。

## 典型的な流れ

1. JSON を貼る
2. 必要なら修復する
3. 整形された構造を確認する
4. 差分や一部の値を抜き出す

## どんなときに便利か

- API の戻り値を確認したいとき
- 設定値の一部だけ見たいとき
- ログの JSON を整えたいとき
- ちょっと壊れた文字列を復旧したいとき

## 使うときのポイント

JSON は、見た目が少し崩れているだけでも意味が分かりにくくなります。
だからこそ、最初に「整える」、次に「比較する」、必要なら「抜き出す」という順に進めると扱いやすいです。

json は、その一連の流れをひとつにまとめるための道具として設計しています。
