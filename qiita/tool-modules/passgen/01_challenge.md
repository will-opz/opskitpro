# パスワード生成器 passgen を作った理由。必要なものをすぐ出せるようにしたかった

OpsKitPro には、DNS やサイト診断のような“調べる”系の機能だけでなく、日常の作業を少し早くする小さなツールも入れています。
その中で `passgen` は、いちばん地味ですが、使う頻度が高いツールのひとつです。

毎回パスワード管理ツールを開くほどではないけれど、
「UUID をひとつ作りたい」「短い PIN をすぐ出したい」「一時利用の文字列がほしい」
という場面は意外と多いです。
そういうときに、余計な設定や画面遷移なしでその場で作れるものがほしかった、というのが出発点でした。

## 何を解決したかったか

- すぐ使えること
- 生成結果をそのままコピーできること
- パスワードだけでなく UUID や PIN も扱えること
- ブラウザ内で完結して、外に送らないこと

パスワード生成器という名前ですが、実際には「短いランダム値を安全に作る道具」として考えています。
用途を 1 つに絞りすぎると、使いたい場面を取りこぼしてしまうからです。

## 既存ツールで感じていたこと

一般的なパスワード生成サイトは便利ですが、機能が多いものほど、
「まずどこを触ればいいか」が少し分かりづらくなります。
逆に、シンプルすぎると今度は用途の幅が足りません。

passgen では、
「よく使うものだけを並べる」ことを優先しました。
高機能さよりも、毎回迷わず同じ手順で使えることを重視しています。

## このシリーズで書くこと

このあと続く記事では、passgen を次の 3 つの視点で整理します。

- UI をどう切ったか
- 生成ロジックをどう分けたか
- 実際にどう使うと速いか

小さなツールですが、こういうものほど設計の差が使い心地に出ます。
後半では、そのあたりをもう少し具体的に書きます。
