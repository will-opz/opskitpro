# passgen の UI 設計。少ない操作で迷わず使えるようにする

passgen は、見た目の派手さよりも「何を押せばいいかがすぐ分かること」を優先して設計しています。
作りたいのは、機能が多い生成サイトではなく、短い作業を素早く終わらせるための作業台です。

## 画面を分けすぎない

この種のツールは、入力・結果・コピーの導線が分かれていれば十分です。
そこで passgen では、

- 左で条件を決める
- 中央で結果を見る
- すぐコピーする

という流れを崩さないようにしています。
モード切り替えを増やしすぎると、使うたびに考えることが増えるからです。

## 標準生成と特殊モードを分ける

通常のパスワード生成、UUID 生成、PIN 生成は、利用頻度も期待する出力も違います。
そのため、同じ画面に詰め込むのではなく、モードを分けて表示しています。

この分け方の狙いは、
- 一度覚えれば次回も迷わないこと
- 入力項目をその場で絞れること
- 出力の意味を誤解しにくいこと

です。

## コピー導線を最優先にする

生成した値は、だいたいそのまま別の場所に貼り付けます。
そのため、結果表示の近くにコピー操作を置き、
生成後の次の動作を 1 回で終えられるようにしています。

こうした小さな設計は目立ちませんが、毎日の体験にはかなり効きます。
UI は“見栄え”より“次の行動までの短さ”で評価される、という前提でまとめました。
