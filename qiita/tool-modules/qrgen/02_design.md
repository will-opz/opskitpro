# qrgen の UI 設計。入力とプレビューだけに絞る

qrgen は、操作を増やさないことを最優先にしています。
入力欄とプレビューが見えていれば、ユーザーはすぐ目的を達成できます。

## 2 段より増やさない

このツールでは、

- 上で内容を入力する
- 下で QR を確認する
- 必要なら保存する

という 3 ステップだけを軸にしています。
オプションを増やしすぎると、QR という単純な用途がかえって重くなるからです。

## プレビューを主役にする

QR は、テキストをそのまま読むのではなく、コードとして見てもらうものです。
だからこそ、プレビューが主役で、入力欄はその補助に置いています。

この配置にすると、
- 入力の結果がすぐ分かる
- 誤入力に気づきやすい
- 保存前の確認がしやすい

という利点があります。

## 保存導線を短くする

実際には、作った QR をそのまま資料やチャットに貼ることが多いです。
そのため、ダウンロードや保存の導線は、プレビューからすぐ触れる位置に置いています。

UI を派手にするより、迷わず終わること。
qrgen はそのための最小構成でまとめています。
