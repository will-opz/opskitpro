# websocket の実装。useWebSocket と複数接続の整理

websocket の実装では、接続の状態管理を見通しよくすることが重要でした。
ひとつの接続だけなら単純でも、複数接続を扱うと更新の責務が増えるからです。

## 接続ごとの責務を分ける

接続状態、受信メッセージ、送信処理は、それぞれ独立して追えるほうが扱いやすいです。
そのため、接続単位のフックと、複数接続を束ねるフックを分ける構成にしています。

## 履歴とテンプレート

WebSocket は同じメッセージを何度も送ることが多いので、
履歴やテンプレートがあると再試行がかなり楽になります。
送った内容をすぐ呼び出せるようにしておくと、調査効率が上がります。

## バイナリ処理

テキストだけでなく、バイナリを扱うケースもあるので、
表示とデータ処理の境界をあらかじめ分けておくと後から拡張しやすくなります。

こういう実装は、接続が安定しているときより、
切断や再接続が起こる場面で差が出ます。
