# websocket の使い方。テキスト送信からバイナリ確認まで

websocket は、接続して終わるツールではありません。
送信、受信、再送、比較までを含めて使うと便利です。

## 典型的な流れ

1. 接続先 URL を入れる
2. 接続状態を確認する
3. テキストメッセージを送る
4. 受信ログを見て応答を確認する
5. 必要ならテンプレートや履歴を使って再送する

## 便利な場面

- ローカルの WebSocket サーバーを確認するとき
- ステージング環境の挙動を見たいとき
- 送受信のイベント順を追いたいとき
- バイナリの扱いを確かめたいとき

## 使うときの注意

接続ツールは、便利な反面、状態が残ることがあります。
再接続や切断のタイミングを意識しながら使うと、トラブルの切り分けがしやすくなります。

websocket は、単純なチェッカーではなく、やり取りを観察するための作業台として使うと真価が出ます。
