# json の設計、実装、用途、使い方

`json` は、開発の中で何度も出てくる「JSON を見たい」「少し直したい」「比較したい」に応えるためのツールです。

## どんな場面で使うか

- API レスポンスを確認したい
- 設定ファイルを整えたい
- ログの JSON を読みやすくしたい
- 一部だけ抜き出したい

## 設計で意識したこと

- 編集と確認を近づける
- 修復、変換、比較を分ける
- 画面を行き来させすぎない

## 実装で意識したこと

- 文字列修復の責務を分ける
- diff や schema の処理を独立させる
- UI 状態とデータ処理を混ぜすぎない

## どう使うとよいか

`json` は、整形ツールというより、JSON を扱うための作業台です。
貼る、直す、比べる、抜き出す、という一連の流れを短くできると、日常の開発がかなり楽になります。
