# OpsKitPro のデザイン原則。UI をあえて控えめにした理由

OpsKitPro の UI は、見栄えよりも「迷わず使えること」を優先しています。
運用ツールは、凝った演出よりも、必要な情報にすぐ届くことのほうが大事だからです。

## 大事にしていること

### 1. 先に結論を見せる

最初に知りたいのは、詳しい説明ではなく「今どうなっているか」です。
結果が先に見えれば、次の行動に移りやすくなります。

### 2. 1 ページ 1 役割

ひとつの画面に役割を詰め込みすぎると、何を見ればいいか分かりにくくなります。
そのため、各ページはできるだけ役割を絞っています。

### 3. 操作は短く、導線は素直に

高頻度で使うものほど、操作は少ないほうがいいです。
「どこを押すか」を毎回考えなくて済む構成を意識しています。

### 4. 余白はあるが、空っぽにはしない

克制的なデザインは、情報を削ることではありません。
不要なノイズを減らしつつ、必要な情報はきちんと残します。

## なぜこの方針なのか

私自身がこのサイトを繰り返し使うからです。
一度開いて、すぐ判断して、すぐ次へ進める。
その体験を壊さないことが最優先でした。

## この原則が影響したもの

- サービス矩陣の見せ方
- ツールページの情報量
- 知識ベースと主サイトの分離
- 多言語化したときの文体

次の記事では、この考え方を工程の収束としてどうまとめたかを書きます。
