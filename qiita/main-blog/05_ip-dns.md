# IP と DNS をどう見せるか。診断しやすい形に変える

IP と DNS は、運用で何度も確認する情報です。
ただし、生の結果をそのまま見せるだけでは、判断しづらいことがあります。

## IP モジュールで見たいもの

IP を調べるときは、単なるアドレスよりも、その背後にある意味を知りたいことが多いです。

- どの地域に見えるか
- どの ASN か
- プロキシ経由に見えるか
- 期待している経路と合っているか

そのため、IP モジュールでは「値」だけでなく「読み取りやすい診断結果」に寄せています。

## DNS モジュールで見たいもの

DNS は、表面上は簡単に見えても、実際には確認観点が多いです。

- 複数 resolver で同じか
- レコードの種類ごとの差はあるか
- 返ってきた値が期待と一致しているか

単発の確認だけでは見落としやすいので、複数の観点から見比べる形にしています。

## 2 つを並べる意味

IP と DNS は別の情報ですが、どちらも「調べたあと、どう判断するか」が重要です。
だからこの 2 つは、単なる検索ツールではなく、診断の入口として並べています。
