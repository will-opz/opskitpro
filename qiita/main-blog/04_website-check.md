# website-check の実装を分解する。並列で調べる理由

`website-check` は、OpsKitPro の中でも特に「今どうなっているか」を早く知りたい場面に向いたモジュールです。
単純な疎通確認ではなく、複数の信号をまとめて見て、判断しやすい形にすることを目指しています。

## 単一チェックでは足りない理由

サイトが見えないとき、原因はひとつとは限りません。

- DNS が正しく引けない
- HTTP が返らない
- TLS / 証明書で止まっている
- CDN やプロキシ層で詰まっている

ひとつの結果だけだと、どこで止まったのか分かりにくいです。
そのため、複数の観点を同時に見ます。

## 実装の考え方

`website-check` では、独立して実行できる確認を並列に走らせています。
そのほうが待ち時間を短くでき、失敗時にもどの層で問題が起きたかを切り分けやすくなります。

- 基礎的な接続
- HTTP の応答
- TLS / 証明書の確認
- 追加の補助信号

## 結果の出し方

重要なのは、失敗しても情報を返すことです。
「失敗した」で終わらせず、どこまで見えて、どこで止まったかを返すほうが、次の判断に役立ちます。

`website-check` は、正否を断定するためというより、排障の初動を速くするためのモジュールとして作っています。
