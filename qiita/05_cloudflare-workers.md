# OpsKitPro を Cloudflare Workers に載せた理由

OpsKitPro は Cloudflare Workers 上で動かしています。  
これは「なんとなく流行っているから」ではなく、プロダクトの性質とかなり相性が良かったからです。

## なぜ Workers なのか

OpsKitPro は、重いアプリというより、すぐ結果を返したい運用ツールです。  
そのため、できるだけ Edge に近い場所で軽く動かしたいと考えました。

Cloudflare Workers を選んだ理由は主に次の 4 つです。

- レスポンスが軽い
- 日本からの体感が良い
- デプロイがシンプル
- 診断系の API と相性が良い

## どんな構成で動かしているか

フロントエンドは Next.js、配信は Cloudflare Workers、ビルドと変換は OpenNext を使っています。  
この構成にすると、アプリの開発体験を保ちながら、Edge 配信の恩恵も受けられます。

## 気をつけたこと

### 1. 重い処理を持ち込まない
Workers は軽さが大事なので、長時間の処理や巨大な状態管理は避けています。

### 2. 外部依存は明確にする
DNS / IP / SSL のような診断は外部サービスに依存しやすいです。  
そのため、どこで失敗したかが分かるようにしています。

### 3. メインサイトと知識ベースを分ける
長文や説明は知識ベースに逃がし、メインサイトはツールと導線に集中させました。

## 結果として良かったこと

- 画面の初速が良い
- 運用ツールとしての体感が軽い
- 更新頻度の高い部分を簡単に回せる
- メインサイトと知識ベースの役割がきれいに分かれた

## まとめ

Cloudflare Workers は、OpsKitPro のような診断系ツールと相性が良いです。  
軽さ、配信の近さ、デプロイの単純さが、そのまま使う人の体感に効いてきます。

OpsKitPro では、Edge 前提の設計にしたことで、かなり扱いやすくなりました。
