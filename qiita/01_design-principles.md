# OpsKitPro の UI をなぜ控えめにしたのか

OpsKitPro を作っていく中で、最初にかなり悩んだのが UI でした。  
最初は少し HUD っぽい、いわゆる「運用画面感」のある見た目にしていましたが、途中からかなり控えめな方向へ寄せています。

今回は、その理由をまとめます。

## 最初は“かっこよさ”を優先していた

プロジェクト初期は、いかにも運用ダッシュボードらしい見た目に寄せていました。

- 暗めの背景
- 強い発光
- 斜体やモノスペースっぽい表現
- 情報量の多いパネル

見た目としては悪くなかったのですが、実際に使ってみると、少し重く感じました。

## 問題は“見た目”ではなく“読みにくさ”

運用ツールで本当に大事なのは、目立つことではなく、すぐ読めることです。

特に日本語 UI では、次のことが重要だと感じました。

- 文字が潰れない
- 余白が足りなすぎない
- 1 画面で主情報と補足情報の差が分かる
- クリックすべき場所が自然に見える

つまり、かっこよさよりも、判断のしやすさを優先するべきでした。

## どう変えたか

### 1. 背景を軽くした
暗さを残しすぎず、カードが自然に浮く浅い背景に寄せました。

### 2. カードを標準化した
ページごとに見た目を変えすぎず、カードの角丸や影、文字サイズを揃えました。

### 3. 強い装飾を減らした
発光や装飾点を減らして、情報が主役になるようにしました。

### 4. 日本語の読みやすさを優先した
日本語では、英語よりも文字幅が大きくなりやすいので、タイトルを短く、説明を自然な長さに揃えました。

## こうして良かったこと

- 結果が読みやすい
- ページ全体の統一感が出る
- 多言語でも崩れにくい
- 運用ツールとしての信頼感が上がる

## まとめ

OpsKitPro の UI は、最初から今の形だったわけではありません。  
むしろ、見た目を削ることで、使いやすさが上がったタイプのプロジェクトです。

「見た目を盛る」より「判断を速くする」ことのほうが、運用ツールでは重要だと改めて感じました。
