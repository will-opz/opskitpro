# Service Matrix を標準化した話。運用ツール一覧の見せ方を揃える

OpsKitPro のサービス一覧は、最初から今の形だったわけではありません。  
ツールが増えるたびに見た目も役割も少しずつばらつき、一覧として読みにくくなっていました。

そこで、普通のツールは標準カードに揃え、Matrix だけ独立した見せ方を保つようにしました。

## 何が問題だったのか

一覧にばらつきがあると、次のことが起こります。

- どれが普通のツールか分かりにくい
- カードごとの情報量が揃わない
- 多言語で見たときに崩れやすい

## どう揃えたか

### 1. カードの見た目を統一した
角丸、余白、影、文字サイズを揃えて、一覧として読めるようにしました。

### 2. 状態表示を統一した
「稼働中」などの表現を揃え、ページごとの差を減らしました。

### 3. 長いタイトルを整理した
日本語で長くなりすぎるタイトルは、意味を変えない範囲で短くしました。

### 4. Matrix だけ特別扱いにした
Matrix は単なる一般ツールではないので、普通のカードに埋もれないようにしています。

## 標準化して良かったこと

- 一覧が読みやすくなった
- ページ全体に統一感が出た
- 多言語対応がやりやすくなった
- 新しいツールを追加しても崩れにくくなった

## まとめ

Service Matrix の標準化は、見た目だけの話ではありません。  
一覧の読みやすさ、拡張のしやすさ、多言語の維持まで含めて、効いています。

OpsKitPro では、普通のものは揃え、特別なものだけ独立させる、という整理に落ち着きました。
